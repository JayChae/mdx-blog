# 간단한 Proof-of-Work 구현

Proof-of-Work는 블록체인의 합의 알고리즘 중 하나로, 계산 작업을 통해 블록의 유효성을 증명합니다.

## PoW의 기본 원리

Proof-of-Work는 다음과 같은 과정을 거칩니다:

1. **난이도 설정**: 목표 해시값의 선행 0 개수
2. **반복 계산**: nonce 값을 증가시키며 해시 계산
3. **목표 달성**: 난이도 조건을 만족하는 해시값 찾기

## 채굴 알고리즘 구현

```javascript
mineBlock(difficulty) {
  while (this.hash.substring(0, difficulty) !== Array(difficulty + 1).join('0')) {
    this.nonce++
    this.hash = this.calculateHash()
  }
  console.log('블록 채굴 완료:', this.hash)
}
```

## 난이도 조정

- **목표 시간**: 블록 생성 간격 (예: 10분)
- **난이도 계산**: 이전 블록들의 생성 시간을 기반으로 조정
- **동적 조정**: 네트워크 해시 파워 변화에 대응

## 보안성과 효율성

PoW의 장단점:

**장점:**

- 검증이 쉽고 안전
- 외부 공격에 강함
- 분산화된 합의 가능

**단점:**

- 높은 에너지 소비
- 확장성 제한
- 51% 공격 가능성
